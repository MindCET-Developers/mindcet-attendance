ארכיטקטורה

תמונת המערכת

לקוח Next.js (Web + PWA) מתקשר עם Supabase לאימות, נתונים וזמן-אמת. הפריסה על Vercel.

💡
בחירת Supabase מספקת אימות Google, מסד Postgres עם RLS, וזמן-אמת — הכול מקופסה אחת, מה שמקצר משמעותית את זמן הפיתוח ל-MVP עבור צוות קטן.

מודל נתונים

סכמת Postgres (ERD)

7 טבלאות עיקריות מקושרות ל-auth.users, עם organizations לבידוד סביבת עבודה (אישי/ארגוני), Views לחישובי דוחות ופוליסות RLS להפרדת הרשאות. ⭐ = שדות/טבלאות שנוספו במצב האישי.

🔒 מדיניות RLS (Row Level Security)

תפקידגישה לרשומות נוכחות
בידוד סביבהכל שאילתה מסוננת לפי organization_id של המשתמש.
Employee / אישיקריאה+כתיבה רק לעצמו — auth.uid() = user_id. כולל עריכת עבר (מסומן is_edited). מצב אישי = סביבה עם חבר יחיד.
Managerקריאה לעובדי המחלקה שלו.
HR / Adminגישה מלאה לפי פוליסה מבוססת role (בתוך הארגון בלבד).

✨ מה כולל המצב האישי

  • Onboarding — בחירת מצב בכניסה הראשונה (אישי / ארגון).
  • החתמה — אותו כרטיס כניסה/יציאה + שעון, מותאם ליחיד.
  • יומן עם סוג יום — עבודה / חופש / מחלה / חג לכל יום.
  • עריכה רטרואקטיבית חופשית של ימים קודמים.
  • דו"ח חודשי — בחירת חודש → תצוגה מקדימה → ייצוא.
  • הגדרות אישיות — שם בדו"ח, שעות יומיות צפויות.

מפת דרכים

8 שלבי יישום

בנייה אינקרמנטלית — כל שלב מספק ערך עצמאי וניתן להדגמה. הזמנים הם אומדן גס לצוות פיתוח קטן.

Phase 0
⏱ ~0.5 שבוע

Setup

scaffold Next.js + Tailwind + shadcn (RTL), פרויקט Supabase, חיבור Vercel.

Phase 1
⏱ ~1.5 שבוע

אימות + Onboarding

Google OAuth דרך Supabase, @supabase/ssr, middleware להגנת routes, מסך בחירת מצב (אישי/ארגון) ויצירת סביבת עבודה.

Phase 2
⏱ ~1 שבוע

Schema + RLS

טבלאות (כולל organizations, day_type), enums, views ופוליסות RLS עם בידוד סביבת עבודה.

Phase 3
⏱ ~1.5 שבוע

החתמה ויומן (משותף)

כניסה/יציאה, יומן אישי, עריכה רטרואקטיבית, סוג יום. בסיס לשני המצבים

Phase 4
⏱ ~1.5 שבוע

Realtime + דשבורד ניהול

"מי נוכח עכשיו", טבלת נוכחות, סינונים. ארגון בלבד

Phase 5
⏱ ~1 שבוע

חופשות

טפסים, אישורים, לוח היעדרויות, יתרות. ארגון בלבד

Phase 6
⏱ ~1 שבוע

דוחות וייצוא

מנוע Excel / CSV / PDF + גרפים, כולל תבנית הדו"ח החודשי האישי.

Phase 7
⏱ ~1 שבוע

מצב אישי (הרכבה)

ניווט ומסכים מותאמים ליחיד, מסך הפקת דו"ח חודשי, הגדרות אישיות. מרכיב מ-3+6

Phase 8
⏱ ~1 שבוע

PWA + ליטוש

manifest, service worker, נגישות, מצבי ריקות ושגיאה.
